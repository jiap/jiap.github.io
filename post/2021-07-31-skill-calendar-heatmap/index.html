制作一个 Github 的贡献图

时间： 2021/07/31   |   分类： skills  

我们可以记录每日的锻炼、每日工作量、每天早起时间等，在年终总结时可以量化。关于贡献图有哪些比较好的呈现方式呢？折线图，柱状图等，这些图并没有给人“眼前一亮”的感觉，想到了 Github 的贡献图——根据算法把贡献量化。

下图给出了记录步数的图[1]。

{% asset_img example.png %}

下面分成两部分讲解如何制作贡献图 (Calendar-heatmap：日历热力图)：

1. 使用 pyecharts 库

使用 pyecharts 的库，在这个库中有日历热力图(Calendar)的介绍[2]。

首先安装这个库，在终端输入

$ python3 -m pip install pyecharts`

教程中给出的是如下代码：

import random
import datetime

import pyecharts.options as opts
from pyecharts.charts import Calendar

begin = datetime.date(2017, 1, 1)
end = datetime.date(2017, 12, 31)
data = [
    [str(begin + datetime.timedelta(days=i)), random.randint(1000, 25000)]
    for i in range((end - begin).days + 1)
]

(
    Calendar(init_opts=opts.InitOpts(width="1600px", height="1000px"))
    .add(
        series_name="",
        yaxis_data=data,
        calendar_opts=opts.CalendarOpts(
            pos_top="120",
            pos_left="30",
            pos_right="30",
            range_="2017",
            yearlabel_opts=opts.CalendarYearLabelOpts(is_show=False),
        ),
    )
    .set_global_opts(
        title_opts=opts.TitleOpts(pos_top="30", pos_left="center", title="2017年步数情况"),
        visualmap_opts=opts.VisualMapOpts(
            max_=20000, min_=500, orient="horizontal", is_piecewise=False
        ),
    )
    .render("calendar_heatmap.html")
)

运行以上 Python 程序，会在该文件夹下渲染出一个 calendar_heatmap.html 的网页文件，即为日历热力图。

  1. 程序中的 data 是一个 List，包含了 2017 年所有的 [日期, 随机数] 的数组元素。
  2. 之后调用 pyecharts 库中的 Calendar 模块，设置热力图的宽度width和高度height，将产生的随机数组 data 赋值给 yaxis_data，在 calendar_opts 中的参数修改热力图的位置。
  3. set_global_opts 中可以调整热力图的名称、colorbar 的起始数值、横纵放置 以及 是否离散。

2. 生成 PNG 图片

现在得到了 calendar_heatmap.html 的网页文件，如果将其放在文章中，可以渲染成图片。这要如何做呢？

2.1 不太有效的方法(踩坑)

网上找到了如何将 html 文件渲染成 png 的方式：

  1. 安装phantomjs:
$ npm install -g phantomjs-prebuilt
  1. 安装pyecharts-snapshost:
$ python3 -m pip install pyecharts-snapshot
  1. 保存图片 在 python 程序中，添加：
from pyecharts_snapshot.main   
import make_a_snapshot  
make_a_snapshot('calendar_heatmap.html', 'calendar_heatmap.png')

但这样做会出bug，查阅了资料：发现问题出在python版本上，据pyecharts (github社区中明确表示上述方法不支持python3.6 3.7等版本。可以使用 snapshot-phantomjs 替代[3-4]。

2.2 可用的方法

根据资料[3-4] 可以使用 snapshot-phantomjs 将 html 文件渲染成 png 文件。

  1. 安装 snapshot-phantomjs 库
$ python3 -m pip install snapshot-phantomjs
  1. 渲染成 png 图片

在 Python 文件中添加

from snapshot_phantomjs import snapshot  
from pyecharts.render import make_snapshot

make_snapshot(snapshot,"calendar_heatmap.html","calendar_heatmap.png")

3. 之后能做的事

在程序中的数据为模拟数据，那我们怎么记录自己的数据呢？可以建一个 csv 文档，然后读取数据。

3.1 CSV 数据 和 读取

建立如下图所示的 CSV 文件(Excel 文件另存为为 CSV)，命名为 exercise.csv

{% asset_img data.png %}
exercise_data = []
with open('exercise.csv', mode='r', encoding='utf-8-sig') as csv_file:
    csv_reader = csv.reader(csv_file, delimiter=',')
    for row in csv_reader:
        ele = [row[0],int(row[1])]
        exercise_data.append(ele)

这里第一列 row[0] 为日期，第二列 row[1] 为真实数据，并添加到 exercise_data。之后再使用 pyecharts 库生成日历热力图。 当然还可以修改 colorbar 的颜色。

3.2 修改 colorbar 的颜色

        .set_global_opts(    
            title_opts=opts.TitleOpts(title=title, pos_left='center'),    
            visualmap_opts=opts.VisualMapOpts(     
                max_=max,      
                min_=0,      
                range_color=['#f5f6f8','#417020'],
                orient="horizontal",
                is_piecewise=False
                )
            )

在 set_global_opts 中添加 range_color 字段，加上 colorbar 的颜色范围。
